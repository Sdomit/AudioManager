# Security policy

## Supported versions

AudioManager is pre-1.0. Security fixes are assessed for the current `main`
branch and the latest published pre-release only.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability and do not include
credentials, private recordings, device identifiers, or proof-of-concept data
that could harm users.

Private vulnerability reporting has not yet been enabled for this repository.
Until it is enabled, email the maintainer at **sarmad.domit@gmail.com** with
`SECURITY` in the subject and include only a concise summary, affected version,
reproduction outline, and impact assessment.

The owner must enable GitHub private vulnerability reporting before the next
official release. Once enabled, this document should be updated to make that
GitHub channel the primary reporting route.

## Disclosure expectations

Please allow time for assessment and a fix before public disclosure. The
maintainer will acknowledge a complete report, assess severity, and coordinate
a disclosure timeline with the reporter when practical.
