# Contributing to AudioManager

## Current contribution status

AudioManager source code is licensed under [Apache-2.0](LICENSE). External code
contributions are not accepted during the current pre-1.0 phase. Please do not
submit a source-code pull request until the owner publishes an updated
contribution-rights policy.

Bug reports, feature suggestions, and documentation feedback remain welcome.
The owner may open this policy later with a DCO, CLA, or another documented
contributor agreement after reviewing the release and support model.

## Feedback and issues

Non-security bugs and feature suggestions are welcome through GitHub Issues.
Before opening one, search existing reports and include the Windows version,
AudioManager version, relevant device configuration, clear reproduction steps,
expected behavior, and observed behavior. Remove personal paths, recordings,
tokens, and credentials first.

For security reports, follow [SECURITY.md](SECURITY.md) instead of creating a
public issue.

## Maintainer workflow

Maintainer changes use a feature or fix branch, a focused pull request, and the
required CI checks. Keep changes reviewable, update documentation when user
behavior changes, and use squash merges for release-bound work once main-branch
protection is configured.
